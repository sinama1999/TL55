# BCG extension for tl55_clean_python

This add-on computes the ballistocardiogram (BCG) waveform from the pressure and flow waveforms generated by the 55-segment arterial tree model.

## What this module does

The BCG is computed from the time derivative of blood momentum in:
- all 55 arterial segments
- the left ventricle

The arterial contribution is evaluated from the average of segment inlet and outlet flow together with each segment's length and anatomical angle. The left-ventricular contribution is evaluated from an approximated left-ventricular volume and outflow velocity term.

## Public API

```python
from tl55 import generate_bcg

bcg = generate_bcg(
    sv_rel=1.0,
    hr_rel=1.0,
    tpr_rel=1.0,
    e_rel=1.0,
    q_input_path="data/Q_inputwave2.mat",
    mv_q_path="data/MV_Q_padded_2.mat",
)
```

The function returns a `BCGResult` object with:
- `bcg_force_N`: filtered BCG waveform
- `time_s`: time axis for the returned beat

## Inputs

The hemodynamic control inputs are the same as for `generate_waveforms`:
- `sv_rel`
- `hr_rel`
- `tpr_rel`
- `e_rel`
- `sl`
- `pad_nodes`
- `l_pad`
- `q_input_path`

Additional input:
- `mv_q_path`: path to the nominal mitral inflow waveform used in the left-ventricular approximation

## Example

```python
from pathlib import Path
import matplotlib.pyplot as plt
from tl55 import generate_bcg

project_root = Path.cwd()

bcg = generate_bcg(
    sv_rel=1.0,
    hr_rel=1.0,
    tpr_rel=1.0,
    e_rel=1.0,
    q_input_path=project_root / "data" / "Q_inputwave2.mat",
    mv_q_path=project_root / "data" / "MV_Q_padded_2.mat",
)

t_plot = bcg.time_s - bcg.time_s[0]

plt.figure()
plt.plot(t_plot, bcg.bcg_force_N, linewidth=2)
plt.xlabel("Time within beat [s]")
plt.ylabel("BCG force [N]")
plt.title("Simulated BCG waveform")
plt.show()
```

## Notes

- The returned BCG waveform is based on the trimmed beat produced by the arterial tree solver.
- The segment angles are read from the arterial data table.
- The output is low-pass filtered at 25 Hz.
- The time axis is trimmed-beat time; for display, it is usually clearer to shift it to start at 0 s.
